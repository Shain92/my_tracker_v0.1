# Описание дашбордов для отслеживания выполнения

## Пример 1: Общий обзор (Overview Dashboard)

**Назначение:** Быстрый обзор ключевых метрик проекта

**Компоненты:**
- **Всего участков** - количество строительных участков
- **Активных проектов** - проекты в работе
- **Выполнение** - средний процент выполнения с прогресс-баром
- **Завершено листов** - количество завершенных проектных листов

**Визуализация:**
- Крупные числа с иконками
- Прогресс-бары для процентных показателей
- Индикаторы изменения (положительные/отрицательные)

## Пример 2: Выполнение по участкам (Site Performance)

**Назначение:** Сравнение эффективности работы строительных участков

**Компоненты:**
- Столбчатая диаграмма с процентом выполнения каждого участка
- Сортировка по проценту выполнения
- Цветовое кодирование (зеленый - высокий, желтый - средний, красный - низкий)

**Метрики:**
- Процент выполнения участка (средний по всем проектам)
- Количество проектов на участке
- Количество завершенных листов

## Пример 3: Топ проектов (Top Projects)

**Назначение:** Отображение проектов с наилучшими показателями

**Компоненты:**
- Список проектов с процентом выполнения
- Название проекта, код и шифр
- Принадлежность к участку
- Статус проекта (завершен, в работе, требует внимания)

**Фильтры:**
- По участку
- По статусу
- По проценту выполнения

## Пример 4: Распределение по статусам (Status Distribution)

**Назначение:** Общее представление о состоянии всех проектных листов

**Компоненты:**
- Количество листов по каждому статусу
- Визуальные бейджи статусов
- Общее количество листов
- Процентное соотношение

**Статусы:**
- Выполнено (зеленый)
- В работе (синий)
- Ожидает (желтый)

## Пример 5: Динамика выполнения (Completion Trend)

**Назначение:** Отслеживание изменения процента выполнения во времени

**Компоненты:**
- Линейный или столбчатый график за выбранный период
- Тренд выполнения
- Прогноз на будущее

**Периоды:**
- Последние 7 дней
- Последние 30 дней
- Последние 3 месяца
- За весь период

## Пример 6: Нагрузка на исполнителей (Executor Workload)

**Назначение:** Мониторинг распределения работы между исполнителями

**Компоненты:**
- Список исполнителей
- Количество активных листов
- Процент выполнения назначенных листов
- Прогресс-бары для каждого исполнителя

**Метрики:**
- Средний процент выполнения
- Количество завершенных листов
- Количество листов в работе
- Количество просроченных листов

## Пример 7: Последние этапы (Recent Stages)

**Назначение:** Отслеживание последних изменений в проектах

**Компоненты:**
- Список последних этапов проектов
- Дата и время создания этапа
- Статус этапа
- Связь с проектом

**Фильтры:**
- По проекту
- По дате
- По статусу
- По автору

## Пример 8: Детальный дашборд проекта (Project Detail Dashboard)

**Назначение:** Детальная информация по конкретному проекту

**Компоненты:**
- Общая информация о проекте (название, код, шифр, участок)
- Процент выполнения проекта
- Список всех проектных листов с их статусами
- График выполнения листов по времени
- История этапов проекта
- Заметки к проекту

## Пример 9: Аналитика по времени (Time Analytics)

**Назначение:** Анализ временных показателей

**Компоненты:**
- Среднее время выполнения листа
- Среднее время выполнения проекта
- Распределение времени по этапам
- График активности по дням недели/часам

## Пример 10: Сравнительный анализ (Comparative Analysis)

**Назначение:** Сравнение показателей между периодами

**Компоненты:**
- Сравнение текущего периода с предыдущим
- Изменение процента выполнения
- Изменение количества завершенных листов
- Тренды по участкам

## Рекомендации по использованию

1. **Общий обзор** - главная страница дашборда, первое что видит пользователь
2. **Детальные дашборды** - для глубокого анализа конкретных участков/проектов
3. **Аналитические дашборды** - для принятия управленческих решений
4. **Оперативные дашборды** - для ежедневного мониторинга

- **Предупреждение:** #f59e0b (желтый)
- **Ошибка:** #ef4444 (красный)
- **Информация:** #3b82f6 (синий)
